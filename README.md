## Overview

FinanceOS is an AI-native personal finance platform that combines bank integrations, automated categorization, and Claude-powered insights into a single, fast, server-rendered application.

**Built with:** Next.js 16 App Router · TypeScript · Supabase · Tailwind CSS · Claude API

---

## Features

- **AI Insights** — Daily and monthly summaries generated by Claude, rendered server-side on first paint
- **Bank Email Parsers** — 6 parsers that extract transaction data from bank notification emails
- **Auto-categorization** — Transactions automatically tagged using Claude Haiku
- **AI Chat** — Conversational interface powered by Claude Sonnet for ad-hoc finance questions
- **AI Review** — Structured spending reviews generated on demand
- **Cron Automation** — 5 scheduled jobs for insight generation, summaries, and data sync
- **Google Calendar Integration** — OAuth-based sync for proactive budget suggestions
- **Admin Panel** — Role-based access control with per-feature model configuration
- **Multi-user Support** — Partner/couples sharing (in progress)

## Architecture

### SSR-First Pattern

All data fetching and AI calls happen on the server. The client receives pre-rendered HTML with no loading spinners or client-side waterfalls.

```typescript
// page.tsx (server component)
export default async function Page() {
  const data = await fetchData(); // Supabase + Claude calls
  return <PageClient initialData={data} />;
}

// PageClient.tsx (client component)
"use client";
export function PageClient({ initialData }) {
  // Zero useEffect fetches — data is ready on first paint
}
```

### Model Selection Strategy

Different features use different Claude models based on complexity and cost requirements:

```typescript
// src/lib/get-user-model.ts
const MODEL_MAP = {
  daily_insight:    "claude-haiku-4-5",   // fast, cheap
  monthly_summary:  "claude-haiku-4-5",
  ai_chat:          "claude-sonnet-4-6",  // balanced, default
  ai_review:        "claude-haiku-4-5",
  auto_categorize:  "claude-haiku-4-5",
};
```

Admins can configure system-level models. Users can only change the chat model.

## Getting Started

### Prerequisites

- Node.js 18+
- A Supabase project
- An Anthropic API key
- (Optional) Google Cloud project for Calendar integration

### Installation

```bash
git clone git@github.com:RahulSaini02/financeos.git
cd financeos
npm install
```

### Environment Variables

```bash
cp .env.example .env.local
```

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
ANTHROPIC_API_KEY=your_anthropic_key
GOOGLE_CLIENT_ID=your_google_client_id         # optional
GOOGLE_CLIENT_SECRET=your_google_client_secret # optional
```

### Database Setup

```bash
npx supabase db push
```

This runs all 14 migrations including schema, RLS policies, and role enums.

### Run Locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## AI Development Methodology

This project was built using a structured AI-native workflow with Claude Code. Key practices:

**Persistent Memory** — Project context lives in `.claude/memory/` so every session starts with full context and zero re-explaining.

**Specialized Subagents** — Four agents with restricted tool access prevent context pollution:

| Agent | Responsibility | Tools |
|-------|---------------|-------|
| `frontend-agent` | UI, components, Tailwind | Read, Edit, Write, Glob |
| `backend-agent` | API routes, Supabase, cron | Read, Edit, Bash |
| `qa-agent` | Validation, writes BUGS.md | Read, Grep, Bash |
| `code-review-agent` | Architecture review | Read, Glob, Grep |

**Custom Skills** — Reusable slash-command behaviors like `/simplify`, `/graphify`, and `/tailwind-responsive-design` encode patterns once and execute consistently.

**Structured Task Management** — All features tracked in `TODO.md` with status, priority, and notes. QA results logged to `BUGS.md`.

Result: ~15–30 minutes per feature from idea to validated code.
